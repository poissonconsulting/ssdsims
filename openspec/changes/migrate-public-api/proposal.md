## Why

The legacy public step functions — `ssd_sim_data.data.frame()`, `ssd_fit_dists_sims()`, and `ssd_hc_sims()` (and therefore `ssd_run_scenario()`, which chains them) — still seed each per-task RNG draw through the **L'Ecuyer-CMRG sub-stream lattice**: a per-`(stream, start_sim)` state advanced with `parallel::nextRNGStream()`/`nextRNGSubStream()` and installed via the `*_state` ops in `R/internal.R`. The targets redesign (`TARGETS-DESIGN.md` §2, §12 roadmap entry `migrate-public-api`) replaces that lattice with a per-task **dqrng + primer** contract: `local_dqrng_state(seed, task_primer(identity))` under a scenario-scoped `local_dqrng_backend()`. That contract is already implemented and tested by `primer-primitives` (archived), which ships the `*_data_task_primer()` seed-and-run wrappers and wires the declarative baseline runner onto them. This step carries the *legacy public surface* over to the same contract, so the whole package shares one RNG path and `cleanup-lecuyer` can later delete the L'Ecuyer machinery wholesale. It is, per the design, **cosmetic and non-gating** for the targets plumbing: it hangs off `primer-primitives` and blocks none of the targets-only steps (`scenario-accessors`, `manifest`, `task-tables`, …), though it does gate `cleanup-lecuyer`.

## What Changes

- Migrate the per-task RNG seeding of `ssd_sim_data.data.frame()`, `ssd_fit_dists_sims()`, and `ssd_hc_sims()` from the L'Ecuyer-CMRG `*_state`/`*_seed` helpers to the **dqrng + primer** contract: each function opens a (reentrant) `local_dqrng_backend()` scope, derives a per-task primer with `task_primer()` over the task's identity, and installs `(seed, primer)` exactly once per task via `local_dqrng_state()` before the state-less op runs — reusing the `*_data_task_primer()` wrappers from `primer-primitives` where their signatures line up (`sample`, `hc`) and the same seed-and-run pattern for the function-valued `fit` step.
- Keep the **public signatures and argument semantics unchanged** (`seed`, `stream`, `start_sim`, `nsim`, function-valued `min_pmix = list(...)`, the fit/hc grids, `.progress`). The reproducibility *mechanism* changes underneath: `stream` becomes a component of the per-task primer identity (distinct `stream` → distinct primer → independent draws) rather than an L'Ecuyer top-level stream. **BREAKING (numeric):** because dqrng `pcg64` is a different generator than L'Ecuyer-CMRG, the actual simulated values, fits, and HCs change for a given `seed`; reproducibility *for a fixed seed* is preserved and is now order-independent. Acceptable pre-1.0 (`0.0.0.90xx`) and intended by the design.
- Preserve the §5 sub-truncation property in the data step: draw the shared `n_max = max(nrow)` sample once per `(stream, sim, replace)` primer and truncate per `nrow` with `head()` (RNG-free), so all `nrow` values share one seeded draw (the existing `slice_sample` prefix behaviour, now expressed on dqrng).
- **Keep the L'Ecuyer `*_seed`/`*_state` helpers as a one-release shim.** `slice_sample_state()`, `fit_dists_state()`, `fit_dists_seed()`, `hc_state()`, `hc_seed()`, and `do_call_seed()` stay defined in `R/internal.R` (still backing the un-migrated generator methods `ssd_sim_data.function`/`.character`/`.fitdists`/`.tmbfit`, the `*-grids` example scripts, and existing tests) but are no longer called by the three migrated functions. `cleanup-lecuyer` removes them.
- Re-express the loop-free reference script `scripts/example-expanded.R` on the new primitives (`sample_data_task_primer()`/`fit_data_task_primer()`/`hc_data_task_primer()`) and re-establish its **byte-equivalence** check against the migrated public path (`ssd_run_scenario()`); reconcile the `example-expanded-grids*.R` variants with the per-task-own-primer model (each task already gets its own primer under dqrng).
- Update Roxygen/`man/` and `GLOSSARY.md` cross-references so the migrated functions document the dqrng + primer seeding (pointing at `task_primer()` / `local_dqrng_state()`), and note the L'Ecuyer path as the deprecated shim.

### Modified Capabilities
- `simulate-data`: the **Reproducible stream-aware seeding** requirement of `ssd_sim_data.data.frame()` moves from L'Ecuyer-CMRG sub-streams to the dqrng + per-task primer contract; `stream` becomes a primer component and the shared `n_max` draw is `head()`-truncated per `nrow`.
- `fit-distributions`: the **Reproducible fitting** requirement of `ssd_fit_dists_sims()` moves to per-task dqrng + primer seeding under a scenario-scoped backend.
- `hazard-concentrations`: the **Reproducible bootstrapping** requirement of `ssd_hc_sims()` moves to per-task dqrng + primer seeding under a scenario-scoped backend.
- `parallel-safe-seeding`: the requirements that bind the *public step functions* to the L'Ecuyer sub-stream lattice (**Non-overlapping sub-streams for simulations**, **Stream isolation**, **Seed independence from caller RNG state**) are re-scoped — the migrated public functions seed via the dqrng `(seed, primer)` contract; the L'Ecuyer sub-stream helpers survive only as the deprecated one-release shim (removed by `cleanup-lecuyer`).

## Impact

- **New code / edits**: per-task seeding rewrites in `R/simulate-data.R` (the `.data.frame` method), `R/fit-dists-sims.R`, and `R/hc-sims.R` to call the `*_data_task_primer()` wrappers / the dqrng seed-and-run pattern with `task_primer()`-derived primers under `local_dqrng_backend()`; retention (marked deprecated) of the `*_state`/`*_seed` helpers in `R/internal.R`. Rewrite of `scripts/example-expanded.R` and reconciliation of `scripts/example-expanded-grids*.R`. Tests in `tests/testthat/test-simulate-data.R`, `test-fit-dists-sims.R`, `test-hc-sims.R` (and a byte-equivalence test mirroring `example-expanded.R`).
- **APIs**: no new exports and no signature changes. Public reproducibility contract shifts from "same `seed`+`stream`+`start_sim` ⇒ identical" (L'Ecuyer) to "same `seed`+task identity ⇒ identical, order-independent" (dqrng). Roxygen/`man/` and `GLOSSARY.md` updated accordingly.
- **Dependencies**: prerequisite `primer-primitives` (archived) supplies `*_data_task_primer()`, `task_primer()`, `local_dqrng_state()`, `local_dqrng_backend()`. No new package dependencies. `dqrng`, `rlang`, `withr` already imported.
- **Downstream**: none gated — the design explicitly makes this non-gating for the targets plumbing. Unblocks `cleanup-lecuyer` (once nothing in the package's active paths uses the L'Ecuyer helpers, only the shim and generator methods remain to remove). The `min_pmix`-by-name resolution used by the declarative path (`fit_data_task()` via `resolve_min_pmix()`) is **not** forced onto the public function, which keeps its function-valued `min_pmix` contract (see design.md).
