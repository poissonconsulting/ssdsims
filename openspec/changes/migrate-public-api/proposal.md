## Why

The legacy public step functions — `ssd_sim_data.data.frame()`, `ssd_fit_dists_sims()`, and `ssd_hc_sims()` (and therefore `ssd_run_scenario()`, which chains them) — still seed each per-task RNG draw through the **L'Ecuyer-CMRG sub-stream lattice**: a per-`(stream, start_sim)` state advanced with `parallel::nextRNGStream()`/`nextRNGSubStream()` and installed via the `*_state` ops in `R/internal.R`. The targets redesign (`TARGETS-DESIGN.md` §2, §12 roadmap entry `migrate-public-api`) replaces that lattice with a per-task **dqrng + primer** contract: `local_dqrng_state(seed, task_primer(identity))` under a scenario-scoped `local_dqrng_backend()`. That contract is already implemented and tested by `primer-primitives` (archived), which ships the `*_data_task_primer()` seed-and-run wrappers and wires the declarative baseline runner onto them. This step carries the *legacy public surface* over to the same contract, so the whole package shares one RNG path. Per the roadmap (`TARGETS-DESIGN.md` §12) it is **cosmetic**: it depends on `primer-primitives` (the contract it reuses) and `scenario-input-types` (its `example-expanded*.R` byte-equivalence re-run must exercise the full public input surface), and blocks none of the targets-only steps (`scenario-accessors`, `manifest`, `task-tables`, …) — but it **gates `cleanup-lecuyer`**: the L'Ecuyer-CMRG helpers and `_seed` shims cannot be removed until the public step functions stop calling them.

## What Changes

- Migrate the per-task RNG seeding of `ssd_sim_data.data.frame()`, `ssd_fit_dists_sims()`, and `ssd_hc_sims()` from the L'Ecuyer-CMRG `*_state`/`*_seed` helpers to the **dqrng + primer** contract: each function opens a (reentrant) `local_dqrng_backend()` scope, derives a per-task primer with `task_primer()` over the task's identity, and installs `(seed, primer)` exactly once per task via `local_dqrng_state()` before the state-less op runs — reusing the `sample_data_task_primer()` / `fit_data_task_primer()` / `hc_data_task_primer()` wrappers from `primer-primitives` **directly** for all three steps.
- Keep the **public signatures unchanged** (`seed`, `stream`, `start_sim`, `nsim`, the fit/hc grids, `.progress`) **except `min_pmix`**: it adopts the name-based pattern of the declarative path — a character vector of function **names** (default `"ssd_min_pmix"`) resolved via `resolve_min_pmix()` — so `ssd_fit_dists_sims()` reuses `fit_data_task_primer()` directly rather than carrying a parallel function-valued path (consistent patterns + code reuse). Custom functions stay usable by name (global-env fallback). **BREAKING (minor API):** `min_pmix` no longer accepts a list of functions.
- The reproducibility *mechanism* changes underneath: `stream` becomes a component of the per-task primer identity (distinct `stream` → distinct primer → independent draws) rather than an L'Ecuyer top-level stream. **BREAKING (numeric):** because dqrng `pcg64` is a different generator than L'Ecuyer-CMRG, the actual simulated values, fits, and HCs change for a given `seed`; reproducibility *for a fixed seed* is preserved and is now order-independent. Acceptable pre-1.0 (`0.0.0.90xx`) and intended by the design.
- Preserve the §5 sub-truncation property in the data step: draw the shared `n_max = max(nrow)` sample once per `(stream, sim, replace)` primer and truncate per `nrow` with `head()` (RNG-free), so all `nrow` values share one seeded draw (the existing `slice_sample` prefix behaviour, now expressed on dqrng).
- **Keep the L'Ecuyer `*_seed`/`*_state` helpers as a one-release shim.** `slice_sample_state()`, `fit_dists_state()`, `fit_dists_seed()`, `hc_state()`, `hc_seed()`, and `do_call_seed()` stay defined in `R/internal.R` (still backing the un-migrated generator methods `ssd_sim_data.function`/`.character`/`.fitdists`/`.tmbfit`, the `*-grids` example scripts, and existing tests) but are no longer called by the three migrated functions. `cleanup-lecuyer` removes them.
- Re-express the loop-free reference script `scripts/example-expanded.R` on the new primitives (`sample_data_task_primer()`/`fit_data_task_primer()`/`hc_data_task_primer()`) and re-establish its **byte-equivalence** check against the migrated public path (`ssd_run_scenario()`); reconcile the `example-expanded-grids*.R` variants with the per-task-own-primer model (each task already gets its own primer under dqrng).
- Update Roxygen/`man/` and `GLOSSARY.md` cross-references so the migrated functions document the dqrng + primer seeding (pointing at `task_primer()` / `local_dqrng_state()`), and note the L'Ecuyer path as the deprecated shim.

### Modified Capabilities
- `simulate-data`: the **Reproducible stream-aware seeding** requirement of `ssd_sim_data.data.frame()` moves from L'Ecuyer-CMRG sub-streams to the dqrng + per-task primer contract; `stream` becomes a primer component and the shared `n_max` draw is `head()`-truncated per `nrow`.
- `fit-distributions`: the **Reproducible fitting** requirement of `ssd_fit_dists_sims()` moves to per-task dqrng + primer seeding under a scenario-scoped backend; the **Default fit configuration** and **Argument validation** requirements change `min_pmix` from a list of functions to a character vector of resolvable names (default `"ssd_min_pmix"`).
- `hazard-concentrations`: the **Reproducible bootstrapping** requirement of `ssd_hc_sims()` moves to per-task dqrng + primer seeding under a scenario-scoped backend.
- `parallel-safe-seeding`: the requirements that bind the *public step functions* to the L'Ecuyer sub-stream lattice (**Non-overlapping sub-streams for simulations**, **Stream isolation**, **Seed independence from caller RNG state**) are re-scoped — the migrated public functions seed via the dqrng `(seed, primer)` contract; the L'Ecuyer sub-stream helpers survive only as the deprecated one-release shim (removed by `cleanup-lecuyer`).

## Impact

- **New code / edits**: per-task seeding rewrites in `R/simulate-data.R` (the `.data.frame` method), `R/fit-dists-sims.R`, and `R/hc-sims.R` to call the `*_data_task_primer()` wrappers / the dqrng seed-and-run pattern with `task_primer()`-derived primers under `local_dqrng_backend()`; retention (marked deprecated) of the `*_state`/`*_seed` helpers in `R/internal.R`. Rewrite of `scripts/example-expanded.R` and reconciliation of `scripts/example-expanded-grids*.R`. Tests in `tests/testthat/test-simulate-data.R`, `test-fit-dists-sims.R`, `test-hc-sims.R` (and a byte-equivalence test mirroring `example-expanded.R`).
- **APIs**: no new exports. One minor signature change — `ssd_fit_dists_sims(min_pmix=)` becomes a character vector of names (was a list of functions). Public reproducibility contract shifts from "same `seed`+`stream`+`start_sim` ⇒ identical" (L'Ecuyer) to "same `seed`+task identity ⇒ identical, order-independent" (dqrng). Roxygen/`man/` and `GLOSSARY.md` updated accordingly.
- **Dependencies**: prerequisites `primer-primitives` (archived; supplies `*_data_task_primer()`, `task_primer()`, `local_dqrng_state()`, `local_dqrng_backend()`) and **`scenario-input-types`** (the `example-expanded*.R` byte-equivalence re-run must exercise the full input surface — `TARGETS-DESIGN.md` §12, `inputs → migrate`). No new package dependencies; `dqrng`, `rlang`, `withr` already imported.
- **Cross-reference — `scalar-ci-flag` and `est-method-setting` (both archived)**: those changes demoted `ci` and `est_method` to scenario-wide simulation settings and removed them from `task_axes("hc")`, pinning that whichever change landed second drop them from the hc primer enumeration. They landed first, so this change's **Reproducible bootstrapping** requirement now keys the hc primer on `(stream, sim, nboot, ci_method, parametric)` only (D6) — no other artifact of either change is touched here.
- **Downstream**: **gates `cleanup-lecuyer`** (`migrate-public-api → cleanup-lecuyer`) — the L'Ecuyer helpers and `_seed` shims can only be removed once the public step functions are off them; no targets step waits on it. Adopting the declarative path's `min_pmix`-by-name resolution (`resolve_min_pmix()`) on the public surface keeps the legacy `ssd_fit_dists_sims()` and the declarative `fit` step on one shared `fit_data_task_primer()`.
