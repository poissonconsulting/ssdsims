## Why

The targets redesign (`TARGETS-DESIGN.md` §2) replaced the original in-memory **monolith** scenario runner and its **L'Ecuyer-CMRG** sub-stream lattice with the per-task **dqrng + primer** contract. That replacement is now *complete and landed*: the declarative + sharded public surface — `ssd_define_scenario()` fed by `ssd_scenario_data()` / `ssd_gen()` (the latter covering the `fitdists` / `tmbfit` / function / function-name generator inputs the monolith accepted), run by `ssd_run_scenario_baseline()` (in-memory) or `ssd_run_scenario_shards()` (Hive-Parquet) — provides every capability the monolith did. The legacy public API (`ssd_run_scenario()` and its `ssd_sim_data()` / `ssd_fit_dists_sims()` / `ssd_hc_sims()` chain) is therefore pure duplication, kept alive only by the L'Ecuyer backend.

This change **retires the legacy public API and removes its implementation and the L'Ecuyer-CMRG RNG in one step**, leaving a single RNG path (dqrng) and a single runner family. It **folds in** what was the separate `migrate-public-api` step: that step's plan to *migrate* the legacy functions onto dqrng is moot once they are deleted rather than ported (the GLOSSARY's one-release "shim" overlap of two RNG backends, anticipated since `dqrng-init` / `local-dqrng-state`, ends here). Because the replacement surface already exists, retirement is a deletion — there is nothing to migrate to that is not already shipped. This is the terminal `cleanup-lecuyer` roadmap step, now self-sufficient.

## What Changes

- **Retire the legacy public API by dropping the monolith runners.** Remove the legacy in-memory scenario path in its entirety:
  - `ssd_run_scenario()` and its five S3 methods (`data.frame`, `fitdists`, `tmbfit`, `character`, `function`) in `R/run-scenario.R`.
  - `ssd_sim_data()` and its five S3 methods in `R/simulate-data.R`.
  - `ssd_fit_dists_sims()` in `R/fit-dists-sims.R` and `ssd_hc_sims()` in `R/hc-sims.R`.
  - The private chainer `run_scenario()` in `R/internal.R` (its only callers are the removed `ssd_run_scenario.*` methods).
  - **BREAKING:** removes the exported generics/methods `ssd_run_scenario()`, `ssd_sim_data()`, `ssd_fit_dists_sims()`, `ssd_hc_sims()`. The supported entry points become `ssd_define_scenario()` + `ssd_run_scenario_baseline()` / `ssd_run_scenario_shards()` (already shipped). The legacy `min_pmix`-as-list-of-functions surface disappears with `ssd_fit_dists_sims()`; the declarative `fit` step already resolves `min_pmix` by name (`resolve_min_pmix()`), so no separate `min_pmix` API change is needed.
- **Remove the L'Ecuyer-CMRG RNG.** Delete the whole legacy RNG backend, now unreachable once the monolith is gone:
  - `R/lecuyer-cmrg-seed.R` in full — the exported scoped helpers `local_lecuyer_cmrg_seed()` / `with_lecuyer_cmrg_seed()` / `local_lecuyer_cmrg_state()` / `with_lecuyer_cmrg_state()` and the internal `get_lecuyer_cmrg_state()` / `get_lecuyer_cmrg_stream_state(s)()` / state helpers (the `parallel::nextRNGStream()` / `nextRNGSubStream()` lattice). **BREAKING:** removes the four exported `*_lecuyer_cmrg_*` helpers.
  - The `*_state` / `*_seed` shims in `R/internal.R`: `slice_sample_state()`, `do_call_seed()`, `fit_dists_state()`, `fit_dists_seed()`, `hc_state()`, `hc_seed()`.
- **NAMESPACE.** Drop the `S3method()` registrations for `ssd_run_scenario` / `ssd_sim_data` and the `export()`s for the monolith runners and the L'Ecuyer helpers (regenerated by `devtools::document()`).
- **Tests.** Remove the monolith/L'Ecuyer suites that exercise only the removed path: `test-run-scenario.R`, `test-simulate-data.R`, `test-fit-dists-sims.R`, `test-hc-sims.R`, `test-lecuyer-cmrg-seed.R`. The dqrng/baseline/shard/targets suites are untouched.
- **Docs and scripts.** Remove the orphaned `man/*.Rd` pages (regenerated), and drop or reconcile the legacy reference/exploration scripts that only drive the monolith path (`scripts/example.R`, `example2.R`, `example-expanded.R`, `example-expanded-grids.R`, `example-expanded-grids-independent.R`, `experiment-substream-restart.R`, `reprex-trace.R` / `reprex-trace_reprex.md`). Strip residual L'Ecuyer references from `GLOSSARY.md`, vignettes, and `TARGETS-DESIGN.md`, and ensure user-facing docs point at the canonical replacement surface (the comprehensive README/vignette overhaul stays the separate `readme` change, now unblocked by this one). Add a `NEWS.md` entry for the BREAKING removals.

## Impact

- **Removed code**: `R/run-scenario.R`, `R/simulate-data.R`, `R/fit-dists-sims.R`, `R/hc-sims.R`, `R/lecuyer-cmrg-seed.R` deleted; the six `*_state`/`*_seed` shims and `run_scenario()` removed from `R/internal.R` (likely emptying it). ~1500–2000 lines of source, tests, and docs.
- **APIs (BREAKING)**: removes eight exported functions — `ssd_run_scenario()`, `ssd_sim_data()`, `ssd_fit_dists_sims()`, `ssd_hc_sims()`, `local_lecuyer_cmrg_seed()`, `with_lecuyer_cmrg_seed()`, `local_lecuyer_cmrg_state()`, `with_lecuyer_cmrg_state()` — plus their `S3method()` registrations and `man/` pages. Acceptable pre-1.0 (`0.0.0.90xx`); the targets redesign is the sole supported path and is already shipped.
- **Dependencies**: none. The replacement public surface is already landed (`scenario-input-types`, `scenario-accessors`, `task-tables`, `shard-runner`, etc., all archived), so there is nothing to wait on. The previously-assumed `mixed-code-lockin` prerequisite is **not needed**: the removed code is not part of any shard's command closure (the step runners never call the monolith or L'Ecuyer functions), so deleting it does not enter any step target's dependency hash and cannot invalidate cached shards — no `tar_cue(depend = FALSE)` pin is required to protect existing runs. **Supersedes `migrate-public-api`** (folded in / removed from the backlog): its migration-then-cleanup two-step is replaced by this single retire-and-remove change. No new package dependencies; once removed, nothing imports `parallel`'s `nextRNGStream`/`nextRNGSubStream` for seeding.
- **Downstream**: terminal cleanup — gates nothing in the targets pipeline, and **unblocks `readme`** (previously *Blocked by `migrate-public-api`*, now gated on this change). After it lands the package has one RNG backend (dqrng) and one runner family; `parallel-safe-seeding` describes only the dqrng path.
