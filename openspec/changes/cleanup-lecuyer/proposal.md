## Why

The targets redesign (`TARGETS-DESIGN.md` §2) replaced the original in-memory **monolith** scenario runner and its **L'Ecuyer-CMRG** sub-stream lattice with the per-task **dqrng + primer** contract that now backs the declarative baseline runner (`ssd_run_scenario_baseline()`), the sharded runner (`ssd_run_scenario_shards()`), and the per-step targets runners. Once `migrate-public-api` retires the legacy public API (the new declarative + sharded surface becomes canonical), the monolith runners and the entire L'Ecuyer RNG that backs them are dead weight: the package would otherwise ship **two runner paths and two RNG backends** indefinitely. This change drops the monolith and everything that becomes unreachable with it, leaving a single RNG path (dqrng) and a single runner family (`ssd_run_scenario_baseline()` / `ssd_run_scenario_shards()` / the targets step runners). This is the long-deferred `cleanup-lecuyer` roadmap step, re-scoped from "remove the L'Ecuyer helpers" to "drop the whole legacy path" in line with the roadmap's `migrate-public-api → retire legacy API` decision.

## What Changes

- **Drop the monolith runners.** Remove the legacy in-memory scenario path in its entirety:
  - `ssd_run_scenario()` and its five S3 methods (`data.frame`, `fitdists`, `tmbfit`, `character`, `function`) in `R/run-scenario.R`.
  - `ssd_sim_data()` and its five S3 methods in `R/simulate-data.R`.
  - `ssd_fit_dists_sims()` in `R/fit-dists-sims.R` and `ssd_hc_sims()` in `R/hc-sims.R`.
  - The private chainer `run_scenario()` in `R/internal.R` (its only callers are the removed `ssd_run_scenario.*` methods).
  - **BREAKING:** removes the exported generics/methods `ssd_run_scenario()`, `ssd_sim_data()`, `ssd_fit_dists_sims()`, `ssd_hc_sims()`. Users run scenarios through `ssd_define_scenario()` + `ssd_run_scenario_baseline()` / `ssd_run_scenario_shards()` (established by the declarative path / `migrate-public-api`).
- **Remove the L'Ecuyer-CMRG RNG.** Delete the whole legacy RNG backend, now unreachable once the monolith is gone:
  - `R/lecuyer-cmrg-seed.R` in full — the exported scoped helpers `local_lecuyer_cmrg_seed()` / `with_lecuyer_cmrg_seed()` / `local_lecuyer_cmrg_state()` / `with_lecuyer_cmrg_state()` and the internal `get_lecuyer_cmrg_state()` / `get_lecuyer_cmrg_stream_state(s)()` / state helpers (the `parallel::nextRNGStream()` / `nextRNGSubStream()` lattice). **BREAKING:** removes the four exported `*_lecuyer_cmrg_*` helpers.
  - The `*_state` / `*_seed` shims in `R/internal.R`: `slice_sample_state()`, `do_call_seed()`, `fit_dists_state()`, `fit_dists_seed()`, `hc_state()`, `hc_seed()`.
- **NAMESPACE.** Drop the `S3method()` registrations for `ssd_run_scenario` / `ssd_sim_data` and the `export()`s for the monolith runners and the L'Ecuyer helpers (regenerated by `devtools::document()`).
- **Tests.** Remove the monolith/L'Ecuyer suites that exercise only the removed path: `test-run-scenario.R`, `test-simulate-data.R`, `test-fit-dists-sims.R`, `test-hc-sims.R`, `test-lecuyer-cmrg-seed.R`. The dqrng/baseline/shard/targets suites are untouched.
- **Docs and scripts.** Remove the orphaned `man/*.Rd` pages (regenerated), and drop or reconcile the legacy reference/exploration scripts that only drive the monolith path (`scripts/example.R`, `example2.R`, `example-expanded.R`, `example-expanded-grids.R`, `example-expanded-grids-independent.R`, `experiment-substream-restart.R`, `reprex-trace.R` / `reprex-trace_reprex.md`). Strip residual L'Ecuyer references from `GLOSSARY.md`, vignettes, and `TARGETS-DESIGN.md` cross-references. Add a `NEWS.md` entry for the BREAKING removals.

## Impact

- **Removed code**: `R/run-scenario.R`, `R/simulate-data.R`, `R/fit-dists-sims.R`, `R/hc-sims.R`, `R/lecuyer-cmrg-seed.R` deleted; the six `*_state`/`*_seed` shims and `run_scenario()` removed from `R/internal.R` (likely emptying it). ~1500–2000 lines of source, tests, and docs.
- **APIs (BREAKING)**: removes eight exported functions — `ssd_run_scenario()`, `ssd_sim_data()`, `ssd_fit_dists_sims()`, `ssd_hc_sims()`, `local_lecuyer_cmrg_seed()`, `with_lecuyer_cmrg_seed()`, `local_lecuyer_cmrg_state()`, `with_lecuyer_cmrg_state()` — plus their `S3method()` registrations and `man/` pages. Acceptable pre-1.0 (`0.0.0.90xx`); the targets redesign is the sole supported path.
- **Dependencies**: prerequisite **`migrate-public-api`** (retires the legacy public API so the monolith is unreachable) and **`mixed-code-lockin`** (the §8.3 shard-pinning recipe, so deleting package source does not force a wholesale shard rebuild). No new package dependencies; once removed, nothing imports `parallel`'s `nextRNGStream`/`nextRNGSubStream` for seeding.
- **Downstream**: terminal cleanup — gates nothing. After this lands the package has one RNG backend (dqrng) and one runner family; `parallel-safe-seeding` describes only the dqrng path.
